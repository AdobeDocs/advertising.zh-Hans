# 协助报表设置

*广告商具有搜索、社交和Commerce点击跟踪以及Adobe Advertising、Adobe Analytics的转化跟踪（具有[!DNL Analytics]集成），或仅在信息源中使用令牌(`ef_id`)提供*

| 选项卡 | 参数 | 描述 |
|----|----|----|
| 不适用 | [!UICONTROL Name] | （可选）报表和模板的名称（如果将报表另存为模板）。 如果应用现有模板，则默认情况下会填充模板名称。 如果不应用模板或输入名称，则默认情况下报表名为`<client name>-<date and time>-<report type>`（如“acme - Apr 3， 2009 11:25:19 AM PDT - Keyword”）。<br><br>您可以选择输入自定义名称，但不要使用文件扩展名。<br><br>如果您正在创建模板以[将报表发送到FTP目录](/help/search-social-commerce/reports/automation/ftp-reports.md)，则可以选择在文件名中的任意位置包含“CSV”（大写字母），以便以CSV格式而不是默认的TSV格式创建文件。 查看发送到FTP目录[&#128279;](/help/search-social-commerce/reports/automation/ftp-reports.md)的报告的文件名要求。 |
|  | [!UICONTROL Save as template] | （除非要根据计划运行报告，否则为可选）将报告设置另存为模板，该模板可在[!UICONTROL Reports] > [!UICONTROL Report Templates]视图中找到，可重复使用以创建新报告。 要将报表另存为模板，请选中该复选框。<br><br>要根据计划运行报表，必须将设置另存为模板。<br><br><b>注意：</b>您可以将当前参数集另存为新模板，即使它基于现有模板也是如此。 |
|  | [!UICONTROL Type] | 要生成的报表类型。 |
| [!UICONTROL Basic  Settings] | [!UICONTROL Template] | （可选）要应用的报表模板，该模板会根据模板预填充报表选项。 将为报表类型保存并且可供您使用的所有模板都将会列出。<br><br>如果选择模板，您仍然可以更改报告选项，甚至可以将报告另存为新模板。 |
|  | [!UICONTROL Date Range] | 要为其生成数据的日期范围：<ul><li><i>[预设范围]：</i>常用时间增量列表，范围从<i>[!UICONTROL Today]</i>到<i>[!UICONTROL Last 180 Days]</i>。 默认值为<i>[!UICONTROL Last 7 Days]</i>，它报告有可用数据的最近七天。 <b>注释：</b> <i>[!UICONTROL Last Month]</i>、<i>[!UICONTROL Last 3 Months]</i>和<i>[!UICONTROL Last 6 Months]</i>显示前一个日历月份的数据。</li><li><i>[!UICONTROL Custom Date Range]：</i>指定开始日期和结束日期。 以YYYY/MM/DD或M/D/YYYY格式输入日期，或单击字段旁边的![日历](/help/search-social-commerce/assets/calendar.png "日历")并选择日期。</li></ul> |
|  | [!UICONTROL Filter By] | （仅限[!UICONTROL Campaign Assist Report]）是否报告特定项目组合或特定广告网络的数据：<ul><li><i>[!UICONTROL Portfolio]</i>（默认）：将营销活动数据包含在一个或多个项目组合中。</li><li><i>[!UICONTROL Search Engine]：</i>要包含一个或多个广告网络中适用于报表类型的促销活动数据。 |
|  | \[主筛选器\] | （仅限[!UICONTROL Campaign Assist Report]）要包含的数据组件。 如果不进行选择，则报表会包含转化路径中发生在任何适用广告网络上的每种事件类型模式的数据行。 您可以选择过滤报告的数据，以便仅在指定组件和子组件中发生系列中的至少一个事件时才包含行。 根据您是按项目组合还是按广告网络进行筛选，请指定要包含的组件：<ul><li><i>[!UICONTROL Portfolio]：</i>一个或多个项目组合或其子组件（营销活动或广告组）。 要选取元件及其所有子元件，请选中元件名称旁边的复选框。 要选择子组件，请选中子组件名称旁边的复选框，然后单击>>以将其移动到[!UICONTROL Selected Filters]列。 例如，要获取Portfolio1及其所有促销活动和广告组的数据，请选中Portfolio1旁边的复选框。 要在Portfolio1的Campaign 1中至少发生一次事件时获取数据，请展开Portfolio1，然后仅选中Campaign 1旁边的复选框。</li><li><i>[!UICONTROL Search Engine]：</i>一个或多个广告网络或其子组件（帐户、营销活动或广告组）。 要选择某个组件及其所有子组件，请选中该组件名称旁边的复选框，然后单击>>以将其移动到[!UICONTROL Selected Filters]列。 要选择子组件，请选中子组件名称旁边的复选框，然后单击>>以将其移动到[!UICONTROL Selected Filters]列。 例如，要在任何[!DNL Google Ads]帐户、营销活动和广告组中至少发生一次事件时获取数据，请选中[!DNL Google AdWords]旁边的复选框。 要仅获取[!DNL Google Ads]帐户1中促销活动1的数据，请展开[!DNL Google Ads]，然后展开帐户1，然后只选中促销活动1旁边的复选框。</li></ul><b>注释：</b><ul><li>要展开列表中的组件（如列出广告网络上的帐户），请单击组件旁边的![向右箭头图标](/help/search-social-commerce/assets/compressed-item.png "向右箭头图标")。</li><li>要查看任何项目的组件类型，请将光标悬停在其上。</li><li>默认情况下，仅列出a)活动和优化项目组合及其活动组件，或b)活动和启用的广告网络帐户、营销活动及其活动组件。 要查看暂停和删除的组件，请单击&#x200B;**[!UICONTROL Show]**&#x200B;旁边的![向下箭头](/help/search-social-commerce/assets/arrow-down-expand.png "向下箭头")并选择&#x200B;**[!UICONTROL All]**。</li><li>在按项目组合生成协助报表时，生成的数据将适用于当前映射到指定项目组合的活动。 该报表不包括在该日期范围内位于项目组合中但尚未存在的营销活动的数据。</li></ul> |
| [!UICONTROL Columns] | [!UICONTROL Use revenue and derived metrics from] | 在报表中填充以下一组默认视图（即使用最小公分母）共有的所有收入和自定义（派生）指标列：<ul><li><i>全部：</i>若要包含在[!UICONTROL Portfolios]和[!UICONTROL Campaigns]视图中所有选项卡的默认视图下共享的列超集，该超集适用于广告商的产品配置。</li><li><i>[!UICONTROL Portfolios]：</i>要包含默认视图为[!UICONTROL Portfolios]视图中的所有选项卡共享的列。</li><li><i>[!UICONTROL Search]：</i>要包含默认视图为[!UICONTROL Campaigns]视图中的所有选项卡共享的列。</li><li><i>[!UICONTROL Display]：</i>已过时</li><li><i>[!UICONTROL Social]：</i>已过时</li><ul><b>注释：</b><ul><li>没有可用的搜索流量量度（例如点击次数或展示次数），并且包含搜索流量量度的派生量度不可用。</li><li>您可以选择删除或重新排列任何可用的收入或派生的量度列。</li><li>尽管可以添加更多列，属性列仍为报表预定义。</li></ul> |
|  | [!UICONTROL Path Size] | 模式中项目（事件类型、关键字或投放位置、广告组或营销活动）的最小和最大数量。 默认路径大小为一(1)个或多个项目以及最多五(5)个项目。 您可以选择仅显示至少包含两(2)个或更多项的路径。 项目类型和项目的最大数量因报告而异：<ul><li>[!UICONTROL Channel Assist Report]：在广告商的[点击回顾窗口](/help/search-social-commerce/glossary.md#c-d)和[展示回顾窗口](/help/search-social-commerce/glossary.md#i-j)内发生的转换路径中最早N个事件的数据逐项列出。 例如，如果选择一(1)个或更多以及最多五(5)个路径的路径大小，则报表将包括最多包含五个事件的转化路径，其中每个跟踪的事件类型模式（如“搜索点击”或“显示展示”）占一行。 路径中最多可以包含30个事件。</li><li>[!UICONTROL Keyword Assist Report]：在广告商的[点击回顾窗口](/help/search-social-commerce/glossary.md#c-d)和[展示回顾窗口](/help/search-social-commerce/glossary.md#i-j)内发生的转换路径中最多可分项列出N个最早搜索关键字或投放位置的数据。 例如，如果选择一(1)个或更多且最多五(5)个路径大小，则报表将包括最多包含五个关键字或版面的路径，并且每个跟踪的关键字字符串或版面模式都有一行。 路径中最多可以包含10个事件。</li><li>[!UICONTROL Campaign Assist Report]：在广告商的[点击回顾窗口](/help/search-social-commerce/glossary.md#c-d)和[展示回顾窗口](/help/search-social-commerce/glossary.md#i-j)内发生的转化路径中，最多可分项列出N个最早营销活动的数据。 例如，如果您选择的路径大小为一(1)个或更多且最多五(5)个，则报表将包括最多包含五个营销活动的路径，其中每个跟踪的营销活动模式占一行。 您最多可以在路径中包含10个营销活动。</li></ul><b>注释：</b><ul><li>要查看协助报表的转化数据，必须添加相应的转化列。</li><li>当转化路径包含的事件类型、关键字或投放位置、广告组或促销活动多于您在报表中逐项统计时，则报表会包含额外的行，用于汇总因项目数量较多导致的转化数据（例如，由10个以上的事件导致的转化有一行）。</li><li>包含具有两个或多个项目的路径时的总数可能低于具有一个或多个项目的路径的总数。</li></ul> |
| [!UICONTROL Columns] | \[报表列\] | 报表中显示的数据列及其顺序：<ul><li>要添加列，请单击左列中的量度名称，然后单击![向右箭头](/help/search-social-commerce/assets/arrow-right-customize.png "向右箭头")。</li><li>若要删除列，请单击右列中的量度名称，然后单击![向左箭头](/help/search-social-commerce/assets/arrow-left-customize.png "向左箭头")。</li><li>要在报表中向左移动列，请单击右列的量度名称，然后单击![向上箭头](/help/search-social-commerce/assets/arrow-up-customize.png "向上箭头")。</li><li>要在报表中将列向右移动，请单击右列中的量度名称，然后单击![向下箭头](/help/search-social-commerce/assets/arrow-down-customize.png "向下箭头")。</li></ul><p><b>注释：</b></p><ul><li>要查看协助报表的转化数据，必须添加相应的转化列。</li><li>要仅列出特定类型的数据，请单击列表上方的任何图标：<ul><li>广告网络帐户或项目组合组件（如[!UICONTROL Campaign Status]）的属性名称和ID的![属性](/help/search-social-commerce/assets/property-columns.png "属性")</li><li>针对为广告商跟踪的转化指标(包括从Adobe Analytics同步的转化和网站参与指标)的![收入指标](/help/search-social-commerce/assets/revenue-metric-columns.png "收入指标")</li><li>由广告商创建的自定义派生量度的![派生量度](/help/search-social-commerce/assets/derived-metric-columns.png "派生量度")</li></ul></li><li>包含许多转化量度的报表，或包含许多转化量度的自定义派生量度的报表，需要更长的时间才能生成。</li><li>要添加、创建或编辑新指标，请参阅“[创建自定义指标](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-create.md)”、“[编辑自定义指标](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-edit.md)”和“[删除自定义指标](/help/search-social-commerce/common-tasks/custom-metrics/custom-metric-delete.md)”。</li><li>有关按报告类型列出的所有可用列的说明，请参阅“[The[!UICONTROL Channel Assist Report]](/help/search-social-commerce/reports/management/assist/channel-assist-report.md)”、“[The [!UICONTROL Campaign Assist Report]](/help/search-social-commerce/reports/management/assist/campaign-assist-report.md)”和“[The [!UICONTROL Keyword Assist Report]](/help/search-social-commerce/reports/management/assist/keyword-assist-report.md)”。</li></ul> |
|  | [!UICONTROL Order Results/Limit Rows by] | 按报表中包含的最多两列对报表进行排序。 每种报表类型的默认值均不同。 要自定义排序顺序，请选择一个报表列，然后选择<i>[!UICONTROL Ascending]</i>（显示从A到Z或从1到100的结果）或<i>[!UICONTROL Descending]</i>（显示从Z到A或从100到1的结果）。 指定至少一个要按其排序的列。 如果按两列排序，则报表将首先按指定的第一列排序，然后按指定的第二列排序。 |
|  | [!UICONTROL Share with others] | 允许有权访问同一广告商数据的其他用户查看生成的报表，并且（如果您将报表另存为模板）使用该模板，但不允许编辑或删除该模板。 默认情况下，不选中此选项。 <b>注意：</b>无论此设置如何，您的报告和模板始终对级别较高的（管理员）Adobe的所有用户以及任何分配的管理员帐户团队成员可见。 |
|  | [!UICONTROL Indicate search engine after entity name] | （仅限[!UICONTROL Campaign Assist Report]）将广告网络名称包含在促销活动名称后面的括号中。 示例： `<campaign name> [Google Adwords]` |
|  | [!UICONTROL Indicate account name after entity name] | （仅限[!UICONTROL Campaign Assist  Report]）将广告网络帐户名称包含在促销活动名称后面的括号中。 示例： `<campaign name> [Google Adwords] [Account1]` |
|  | [!UICONTROL Indicate event  type after entity name] | （仅限[!UICONTROL Campaign Assist Report]）在促销活动名称后的方括号中包含事件类型。 示例： `<campaign name> [click]`或`<campaign name> [Google Adwords] [Account1] [impression]` |
| [!UICONTROL Advanced Filters] | \[高级过滤器\] | （仅限[!UICONTROL Campaign Assist Report]）仅在量度的值满足指定条件时返回行；该量度不需要作为列包含在报表中。 可用量度的列表因报告类型而异，但可能包括广告商的自定义派生量度、每个搜索引擎和组合组件（例如[!UICONTROL Campaign ID]和[!UICONTROL Campaign Status]）的ID和属性名称、广告商的转化量度，以及广告网络中与点击相关的量度。 可用的运算符包括<i>[!UICONTROL contains]</i>、<i>[!UICONTROL starts with]</i>、<i>[!UICONTROL equals]</i>、<i>[!UICONTROL is greater than]</i>、<i>[!UICONTROL is greater than or equal to]</i>、<i>[!UICONTROL is less than]</i>、<i>[!UICONTROL is less than or equal to]</i>或<i>[!UICONTROL isn't equal to]</i>。<br><br>要应用一个或多个筛选器，请执行以下操作：<ul><li>选择量度和运算符，然后输入适用的值。 例如，要仅返回点击次数超过100次的关键字，请选择[!UICONTROL Clicks]，选择[!UICONTROL >]，然后在输入字段中输入100。</li><li>（要应用其他筛选器）对于每个其他筛选器，请单击&#x200B;**[!UICONTROL +Add Filter]**，选择&#x200B;**[!UICONTROL AND]**&#x200B;或&#x200B;**[!UICONTROL OR]**，选择量度和运算符，然后输入适用的值。</li></ul> |
| [!UICONTROL Scheduling and Delivery] | [!UICONTROL Report Schedule] | （可选；仅在选择了“[!UICONTROL Save as template]”选项时可用）何时运行报告： <i>[!UICONTROL Now]</i> （运行报告一次；默认）、<i>[!UICONTROL Daily]</i>、<i>[!UICONTROL Weekly on] [周中某天]</i>或<i>[!UICONTROL Every Month] [月中某天]</i>。 对于<i>[!UICONTROL Now]</i>以外的所有时间段，选择广告商时区中的小时，从上午9:00开始。 |
|  | [!UICONTROL Email Recipients] | <b>注意：</b>此设置仅在[!UICONTROL Notification Center][&#128279;](/help/search-social-commerce/notifications/notification-edit.md)内启用[!UICONTROL Reports]的电子邮件通知时使用。<br><br>注册的Search、Social和Commerce用户的电子邮件地址，当报告完成或由于错误被取消时，将向其发送通知。 默认情况下，会输入用户帐户的地址。 要指定多个地址，请用逗号、空格或新行分隔它们。 如果计划报告重复运行，则每次完成报告时都会发送通知。 |
|  | [!UICONTROL Email Notification] | <b>注意：</b>此设置仅在[!UICONTROL Notification Center][&#128279;](/help/search-social-commerce/notifications/notification-edit.md)内启用[!UICONTROL Reports]的电子邮件通知时使用。<br><br>（当指定[!UICONTROL Email Recipients]时）要包含在发送给任何指定地址的电子邮件通知中的内容：<ul><li><i>[!UICONTROL Notification Only]</i>（默认）：仅发送报告完成或失败的通知，不带附件。 该通知包括所有报表格式的临时下载链接。</li><li><i>[!UICONTROL XLS Attachment]：</i>如果文件小于约10 MB，则以XLS格式包括已完成报表的副本。 超过1 MB的文件将被压缩。</li><li><i>[!UICONTROL TSV Attachment]：</i>如果文件小于10 MB，则以TSV格式包括已完成报表的副本。 超过1 MB的文件将被压缩。</li><li><i>[!UICONTROL CSV Attachment]：</i>如果文件小于10 MB，则以CSV格式包括已完成的报表的副本。 超过1 MB的文件将被压缩。 |
